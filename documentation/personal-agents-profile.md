# Personal AGENTS Profile

This is a concise long-term working-style profile derived from the imported setup. It intentionally excludes client-specific and temporary project detail.

## Development Preferences
- Read the repo rules first, then act
- Prefer existing architecture over novelty
- Keep edits narrow and defensible
- Favor TypeScript, explicitness, and predictable structure
- Validate important changes with actual commands, not assumptions

## Web Design Preferences
- Modern, premium, restrained interfaces
- Strong typography, spacing, and motion craft
- Real content over filler
- White-dominant layouts over noisy visual systems
- Motion should clarify hierarchy, not decorate aimlessly

## UI/UX Preferences
- Mobile-first
- High visual polish without clutter
- Familiar interaction patterns
- Strong contrast and focus states
- Clear CTAs and fast scanning
- Accessibility and responsiveness are baseline requirements

## AI Coding Workflow Preferences
- Inspect first
- Build from repo context, not generic patterns
- Use specialized skills when the task matches them
- Increase reasoning effort only when complexity warrants it
- Prefer concrete implementation over abstract advice

## Documentation Preferences
- Keep instructions concise and operational
- Separate stable preferences from project-specific rules
- Use summary docs to reduce duplication
- Preserve source-of-truth files rather than rewriting everything into one place

## Git Workflow Preferences
- Avoid broad, unrelated refactors
- Do not overwrite user changes casually
- Keep changes reviewable
- Verify before concluding work
- Treat deployment, data, and infra changes as higher-risk operations
